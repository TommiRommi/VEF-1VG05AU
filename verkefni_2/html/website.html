List of things not to do on a embedded machine or do it anyway i don't careor do i?no i don't :)

  • Program should never execute outside of program space.
      • The program can't stop, it will continue to execute forever and ever. To stop a program from causing a disaster, the program counter needs to be trapped in a endless loop.
  • Never run undocumented opcodes unless you know what you are doing.
      • Running undocemented opcodes can lead undefined code behavior and bugs. In the worst case undefined opcodes can lock up the CPU, cause a bus overload, and jam the pipeline
  • Never use a computer component without it being properly initlized.
      • most components on a computer need to be initlized before you can properly use them, unless it can cause undefined behavior. Initlization is done via something know as bootcode/IPL or for more complicated machines BIOS. Refer to the components datasheet for more info.
